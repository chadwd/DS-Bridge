# Agent Journals

Work journals for DS-Bridge contributors. Continuous logs of development progress, decisions, and learnings.

## File Naming Convention

```
{contributor}-agent-work-journal.md
```

**Examples:**
- `chad-agent-work-journal.md` - Chad's work journal
- `jake-agent-work-journal.md` - Jake's work journal

## Journal Purpose

Journals capture:
- Daily development progress
- Technical decisions and rationale
- Problems encountered and solutions
- Ideas and future considerations
- Links to commits and PRs

## Best Practices

1. **Date entries** - Start each entry with date header
2. **Link commits** - Reference git commits for traceability
3. **Capture decisions** - Document why, not just what
4. **Note blockers** - Record issues for future reference
